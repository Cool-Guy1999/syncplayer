SyncPlayer is an android client application for connecting to SyncPlay compatible servers. The app creates a direct connection to the servers that you, the user, specifies. This privacy policy will cover everything that the app has access to and how the data is used.

Last updated: 7th May 2017

1.0 Scope of the policy

This policy only applies to the most recent version of this app that is distributed on the Google Play Store. Users who have opted for the beta or alpha versions of the app on the Google Play Store are also subject to this policy.

1.1 Data required for core functionality

  • Syncplay server hostname/ip (user specified endpoint)
  • Syncplay ‘Room’ details
  • Filename (or the URL) of the selected media file, corresponding metadata that includes its file size and duration (playback length)
  • Syncplay password for ‘Room’ (optional)

The room, nickname, file name (or the URL) of the media file you picked, its duration and file size are all transferred over to the user specified endpoint (hostname/ip) as plain human readable JSON strings over TCP/IP. Passwords (if any) are MD5 hashed.
Any media player states such as current position, pause, play etc are also transferred in a similar manner.

Encryption is not supported because of the limitation of syncplay protocol.

1.2 Permissions used by the app

The app currently requires the following permissions:-

INTERNET, ACCESS_NETWORK_STATE: This is required for core app functionality of connecting to syncplay servers.

READ_EXTERNAL_STORAGE: This is used for loading the media file from the phone. The app only processes information on the file you select and any information retrieved is handled as mentioned in Section 1.1

1.3 Changes to this policy

This privacy policy may change at any time (more likely when a new update to the app has been issued) and the updated policy shall be reflected on the Google Play store’s ‘privacy policy’ link.
